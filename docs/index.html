Why DemoScript?

YAML-Defined

Write demos in simple YAML. Version control friendly, easy to review, easy to maintain.

Live API Execution

Execute real REST calls and shell commands. No screenshots or mocked responses.

Visual Effects

Confetti, sounds, transitions, neon glow effects. Make your demos memorable.

OpenAPI Integration

Auto-generate forms from your OpenAPI spec. Point to your docs and go.

Dashboard & Navigation

Built-in dashboard with health checks, data cards, and sidebar navigation.

Open Source

MIT licensed. Run locally, extend freely, contribute back.

Simple YAML Syntax

Define your demo steps in a familiar format

demo.yaml
title: "My API Demo"
description: "Interactive walkthrough of our API"

settings:
  base_url: "https://api.example.com"
  effects:
    confetti: true

steps:
  # Markdown slides
  - slide: |
      # Welcome to our API
      Let's explore the endpoints together.

  # REST API calls with auto-generated forms
  - rest: POST /users
    title: "Create a User"
    form:
      - name: email
        default: "demo@example.com"
    save:
      userId: response.id

  # Use saved variables
  - rest: GET /users/$userId
    title: "Fetch the User"

Get Started

$ npm install -g @demoscript/cli

Then run demoscript serve your-demo/ to start presenting
